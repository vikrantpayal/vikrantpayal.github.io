Role based access control

The three main components when setting up RBAC in K8s are:
  1. The subject: Groups or users or service accounts
  2. The API resource : ConfigMap, Pod, Deployment, Node
  3. The Operations : Create, List, Watch, Delete


How do I create a service account?
kubectl create serviceaccount < service-account-name >


Here is the declarative way to do it.
apiVersion: v1
kind: ServiceAccount
metadata:
    name: build-bot

How do I get the details of any service account?
kubectl describe serviceaccount < service-account-name >


To list all service accounts
kubectl get serviceaccounts
